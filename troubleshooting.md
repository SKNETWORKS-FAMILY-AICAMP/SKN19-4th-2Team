# 🚀 트러블 슈팅

## 프론트 엔드

### 1. **메인 페이지 스크롤 문제**

- **문제:** 스냅 기능으로 인한 스크롤 딜레이 및 **Features Section**이 **navbar** 뒤로 숨겨짐.
- **원인:** `main_script.js`에서 스크롤 위치 계산 시 **navbar** 높이를 고려하지 않음.
- **해결 방법:**
    - **CSS scroll-snap 비활성화:** `scroll-snap-type: none;`
    - **JavaScript 수정:** 스크롤 위치 계산 시 `navbar` 높이 빼기 (`offsetTop - navbarHeight`).
    - **scroll-padding-top 조정:** 스냅 기능과 함께 작동하도록 값 맞추기.
- **선택한 해결책:** `main_script.js`에서 `navbar` 높이를 고려하여 스크롤 위치 계산.

### 2. **회원정보 수정: 별명과 비밀번호 분리**

- **문제:** 별명과 비밀번호 변경이 하나의 폼에 통합되어 있어 불편.
- **해결 방법:**
    - **HTML 수정:** 별명 변경 폼과 비밀번호 변경 폼을 각각 독립적인 `<form>` 태그로 분리.
    - **views.py 수정:** 별명과 비밀번호 변경 폼을 개별 처리 로직으로 분리.
    - **보안:** 비밀번호 변경 후 폼 자동 초기화.
    - **사용자 경험:** 성공/실패 메시지 명확히 표시.
- **주요 개선점:**
    - 별명만 수정 시 비밀번호 입력 불필요.
    - 비밀번호만 수정 시 별명 입력 불필요.
    - 각각 독립적인 유효성 검증 및 피드백 제공.
